# Purpose

This document defines the core gameplay mechanics of Anu Sabi. It explains how the game is played, the player experience, game flow, and the rules that govern each session.

---

# Gameplay Overview

Anu Sabi is a multiplayer social party game where players respond to prompts, create humorous or creative answers, and interact through discussion, voting, and laughter.

There are no objectively correct answers.

The goal is to entertain, surprise, and outsmart other players through creativity.

Each match consists of multiple rounds where players answer prompts generated by the Rambler Engine or selected from themed content packs.

---

# Core Gameplay Pillars

Every gameplay mechanic should support one or more of these pillars.

## Social

Players should constantly interact with each other.

## Creative

Players are encouraged to invent unique answers.

## Funny

Unexpected outcomes are desirable.

## Simple

Rules should be easy to understand.

## Replayable

Every session should feel different.

# Game Modes

## Classic Mode

The standard party experience.

Players answer prompts.

Players vote.

Points are awarded.

Winner is determined after several rounds.

---

## Quick Play

Shorter matches.

Fewer rounds.

Ideal for casual play.

# Match Structure

A match consists of:

1. Lobby
2. Category Selection
3. Prompt Generation
4. Player Responses
5. Reveal Phase
6. Voting Phase
7. Score Update
8. Next Round
9. Final Results

---

# Player Actions

Players may:

- Join rooms
- Create rooms
- Select avatars
- Submit answers
- Vote
- React to answers
- Earn rewards
- Unlock content
- View rankings
- Share results

---

# Prompt Types

Examples include:

- Finish the sentence.
- What happens next?
- Who would most likely...?
- Caption this.
- Roleplay.
- Story continuation.
- Debate.
- Would you rather?
- Mystery scenario.
- Random challenge.

The Rambler Engine may combine multiple prompt styles into a single round.

---

# Scoring System

Players earn points through:

- Votes received
- Bonus achievements
- Winning rounds
- Daily objectives
- Special events

Additional scoring mechanics may be introduced in future updates.

---

# Win Conditions

A player wins by accumulating the highest score after all rounds.

Alternative win conditions may exist in future game modes.

---

# Difficulty

Difficulty is not based on skill.

Instead, replayability comes from:

- Random prompts
- Player creativity
- Dynamic combinations
- Social interaction
