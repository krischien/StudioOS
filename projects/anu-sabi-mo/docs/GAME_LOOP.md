# Anu Sabi
## Core Game Loop

Version: 1.0
Status: Living Document
Last Updated: February 2026

---

# Purpose

This document defines the complete gameplay loop of Anu Sabi, from launching the game to completing a match and returning for future sessions. It serves as the blueprint for player engagement, retention, and long-term replayability.

---

# Design Goals

The game loop should:

- Be easy to understand.
- Encourage repeated play sessions.
- Reward creativity over correctness.
- Promote social interaction.
- Keep downtime between actions minimal.
- Create memorable moments every match.

---

# High-Level Game Loop

```
Launch Game
      ↓
Home Screen
      ↓
Choose Game Mode
      ↓
Join or Create Lobby
      ↓
Room Setup
      ↓
Match Starts
      ↓
Prompt Appears
      ↓
Players Think
      ↓
Players Submit Answers
      ↓
Answers Revealed
      ↓
Players Vote / React
      ↓
Round Results
      ↓
Leaderboard Update
      ↓
Next Round
      ↓
Final Results
      ↓
Rewards & Progress
      ↓
Play Again or Return Home
```

# Detailed Player Journey

## 1. Launch

Player opens the app.

Possible actions:

- Continue previous session
- Daily rewards
- Event announcements
- Friend invitations

---

## 2. Home Screen

Player may:

- Play
- View Profile
- Customize Avatar
- Browse Content Packs
- View Achievements
- Check Daily Challenges
- Visit Shop (Future)

---

## 3. Select Game Mode

Available modes:

- Classic
- Quick Play
- Custom Room

Future modes:

- Ranked
- Tournament
- Story Mode
- Community Events

---

## 4. Lobby

Players:

- Join friends
- Invite others
- Configure match settings
- Chat (optional)
- Ready up

---

## 5. Match Begins

The Rambler Engine generates:

- Prompt
- Category
- Modifiers
- Random twists (optional)

Countdown begins.

---

## 6. Thinking Phase

Players create responses.

Good prompts should encourage:

- Humor
- Creativity
- Unexpected ideas
- Storytelling

---

## 7. Submission Phase

Players submit answers.

Options may include:

- Edit before timer ends
- Confirm submission
- Early ready bonus (future)

---

## 8. Reveal Phase

Responses appear.

Depending on game mode:

- Random order
- Anonymous
- Player names hidden until voting ends

This prevents bias.

---

## 9. Voting Phase

Players vote for:

- Funniest
- Most Creative
- Most Unexpected
- Best Overall

Future updates may support multiple voting styles.

---

## 10. Results

Display:

- Round Winner
- Points Earned
- Bonus Awards
- Funny Highlights

Animations should feel rewarding without slowing gameplay.

---

## 11. Leaderboard

Update:

- Current rankings
- Player streaks
- Round history

Keep this screen brief.

---

## 12. Next Round

Repeat the loop until:

- Maximum rounds reached
- Match ends early
- Players leave

---

## 13. Match Complete

Show:

- Winner
- MVP Awards
- Fun Statistics
- Favorite Answers
- Match Summary

Allow players to:

- Play Again
- Share Results
- Return Home

---

# Reward Loop

Players receive rewards through:

- XP
- Coins (future)
- Daily Missions
- Achievements
- Event Rewards
- Content Unlocks

Rewards encourage continued play without requiring purchases.

# Replayability Drivers

Every match should feel different because of:

- Different players
- Random prompts
- Rambler Engine combinations
- Voting outcomes
- Content Packs
- Future seasonal events

---

# Friction Points to Avoid

Avoid:

- Long waiting times
- Complicated rules
- Excessive menus
- Repetitive prompts
- Slow animations
- Too much reading
- Forced tutorials

---

# Success Metrics

A healthy game loop should improve:

- Match completion rate
- Average session duration
- Consecutive matches played
- Daily active users
- Player retention
- Player invitations
- Shared match results
