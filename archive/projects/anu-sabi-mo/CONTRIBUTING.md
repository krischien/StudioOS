# 🤝 Contributing to Anu Sabi

Thank you for your interest in contributing to Anu Sabi.

Our goal is to build a high-quality, maintainable, and well-documented project.

---

# Guiding Principles

- Quality over speed.
- Documentation is part of development.
- Keep changes focused.
- Prefer simple solutions.
- Respect existing architecture.

---

# Development Workflow

1. Discuss major changes before implementation.
2. Update documentation when functionality changes.
3. Keep commits focused on a single purpose.
4. Test before submitting changes.

---

# AI-Assisted Development

This project uses AI as a collaborative tool.

Typical workflow:

- Kristian Arendain → Product decisions
- Cursor → Implementation support
- Ziv (ChatGPT) → Documentation and architecture

Human approval is required for all significant decisions.

---

# Coding Standards

- Use TypeScript best practices.
- Favor reusable React components.
- Keep Tailwind classes readable.
- Avoid unnecessary complexity.
- Write self-explanatory code.

---

# Documentation Standards

Every major feature should include:

- Documentation updates
- Related roadmap changes (if applicable)
- TODO updates (if applicable)
- Changelog entry (for releases)

---

# Thank You

Every contribution helps improve Anu Sabi.
