# 📚 Content Documentation

> **Defines how game content is designed, written, reviewed, localized, maintained, and expanded for Anu Sabi.**

---

# Purpose

The Content Documentation describes the complete lifecycle of game content within Anu Sabi.

While the Game Design documentation explains how the game works, this section explains how the game's content is created and maintained.

The goal is to ensure that every phrase, category, challenge, and future content pack follows a consistent standard of quality.

---

# Scope

This documentation covers:

- Phrase creation
- Writing standards
- Difficulty guidelines
- Category organization
- Editorial review
- Localization
- Content releases
- AI-assisted content generation
- Rambler Engine content
- Analytics
- Content maintenance

It serves as the primary reference for anyone responsible for creating or managing game content.

---

# Objectives

The content system should ensure that all game content is:

- Fun
- Fair
- Consistent
- High quality
- Culturally appropriate
- Easy to maintain
- Easy to expand

Every piece of content should contribute positively to the overall player experience.

---

# Content Lifecycle

Every piece of content follows a structured lifecycle.

```
Idea

↓

Writing

↓

Review

↓

Testing

↓

Approval

↓

Release

↓

Player Feedback

↓

Revision

↓

Archive (if needed)
```

This workflow helps maintain quality as the game grows.

# Guiding Principles

Every piece of content should be:

- Easy to understand
- Enjoyable to solve
- Appropriate for the target audience
- Consistent with the game's tone
- Free from unnecessary ambiguity
- Expandable for future updates

Quality should always take priority over quantity.
